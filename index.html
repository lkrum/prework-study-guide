HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The DOCTYPE html declaration tells the browser what document type it.
  • The html opening tag represents the start of our HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.

CSS

  • Inline styles apply to a specific HTML tag.
  • A margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • The Wild card symbol * is used to apply a rule to all the elements visible on the page.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
  • The block value assigns each image to start on a new line.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The period preceding card designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS.

Git

  • git branch: Lists your branches
  • git status: Checks what branch we are currently on
  • git checkout -b branch-name: Creates a new branch and switches to it
  • git add -A: add command adds modifications in the current working branch to the staging area; -A flag indicates that we want to add all of the changes
  • git push: Creates a new remote branch or add new changes to an existing branch
  • git pull: Fetch and merge any commits from the tracking remote branch
  • git commit -m: Commits your staged content as a new commit snapshot
  • git pull origin [name]: pull command receives a branch's modifications into the local environment; the origin keyword indicates the source of the pull will be in the GitHub repo; the main keyword indicates the branch. In our case, we are receiving a branch called main from the GitHub repository.

JavaScript

  • Control flow: The order in which a computer executes code in a JavaScript file
  • Variables: Containers that store values (keyword: var)
  • Comments: Snippets of text that can be added along with code. The browser ignores text marked as comments
  • Conditionals: Code structures used to test if an expression returns true or not
  • Operator: A mathematical symbol that produces a result based on two values (or variables)
  • Control flow: The order in which a computer executes code in a script
  • Array: A single variable used to hold a group of related data
  • Index: The number that signifies the location of a data item in an array
  • Commenting out code makes it inactive without deleting it
  • A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy.
